# Comprehensive MLOps Project Specification: Neural vs. Conventional TSP Solver

## 1. Elaborated Problem Statement

### The Business Context
The Traveling Salesman Problem (TSP) is a foundational algorithmic challenge with massive real-world implications in logistics, supply chain management, delivery routing, and microchip manufacturing. The objective is simple to state but notoriously difficult to solve: *Given a list of cities and the distances between each pair, what is the shortest possible route that visits each city exactly once and returns to the origin city?*

### The Technical Challenge
TSP is an NP-hard problem. As the number of cities ($N$) increases, the number of possible routes grows factorially ($O(N!)$), making exact brute-force computation physically impossible for large datasets. Historically, the industry has relied on **Exact Solvers** (which are slow but guarantee the best path) and **Heuristics** (like Nearest Neighbor or Google OR-Tools, which are fast but provide approximations). 

Recently, Deep Learning—specifically **Transformer architectures** and Graph Neural Networks—has emerged as a novel way to approximate solutions by "learning" routing strategies from historical data rather than computing them purely algorithmically.

### Project Objective
The goal of this project is to build an end-to-end, zero-cost MLOps platform that directly compares conventional algorithmic solvers against a trained Transformer neural network. 

The platform must provide:
1. **Interactive Visualization:** A web interface allowing users to generate random city nodes and visually observe the route generated by different algorithms.
2. **Performance Benchmarking:** Real-time metrics comparing the execution latency and total path distance of the Neural model vs. Conventional heuristics.
3. **Automated MLOps Pipeline:** A production-ready CI/CD architecture that automatically tests and deploys the inference API using free-tier cloud resources.

## 2. System Architecture & Tech Stack

- **Frontend (UI & Rendering):** HTML5 Canvas, JavaScript, CSS (Hosted statically on GitHub Pages).
- **Backend (API & Inference):** Python, FastAPI, Uvicorn (Hosted in Docker on Hugging Face Spaces).
- **Machine Learning Engine:** PyTorch (for the Transformer model).
- **Conventional Engine:** SciPy, NetworkX, Google OR-Tools.
- **Database (Experiment Logging):** MongoDB Atlas Free Tier (via `pymongo`).
- **Package Management:** `uv` (for rapid Python dependency resolution).
- **CI/CD Pipeline:** GitHub Actions.
